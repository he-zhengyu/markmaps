# LLM Self-Correction 課程筆記

## 📌 課程主題總覽
- 核心議題：大型語言模型 **Self-Correction 自我修正能力**
- 定義：無人為介入，模型自主發現錯誤、修正輸出
- 課程脈絡：2023 模型自省能力初探 → 2024 Reasoning 延伸 → 2025 最新進展
- 三大技術方向
  - 修改 Inference 推論過程
  - 優化執行 Workflow 流程
  - 直接優化模型參數 / 內在推理機制

## 🔹 方向一：修改 Inference 推論過程
- 核心邏輯：不改模型參數，生成階段做錯誤偵測+修正
### 1. 錯誤訊號抽取基礎研究
- 透過模型中間 Representation 區分正確/錯誤輸出
- 二元分類器：基於隱藏層特徵預測答案正誤
- True Facts 論文：正確/錯誤表征向量相減，差值向量修正輸出
- 局限：需額外標註數據、預先蒐集好壞表征

### 2. Contrastive Decoding 對比解碼
- 核心思想：構造「模型必然答錯」的對照輸入，向量差值修正結果
- 通用公式：最終向量 = 原向量 ± α×錯誤對比向量
- 優點：訓練無關、純推論階段套用
- 缺點：雙份推論、額外算力消耗

### 3. 主流衍生技術分支
- **原始 Contrastive Decoding（2022）**
  - 小模型輸出作為錯誤對照
  - 基於 Logits/機率分布相減修正
- **DoLa（Layer Contrasting Layers）**
  - 基於 Logit Lens 技術：取用模型淺層輸出作為錯誤參照
  - 無需額外模型、極低運算開銷
  - 已內建於 Hugging Face Transformer
- **Layer CD**
  - 跨層對比：視覺編碼器深層vs淺層表征
  - 解決模型先驗知識干擾細節判斷問題
- **ICD 指令對比解碼**
  - 加入「降智咒語」提示，強制模型輸出錯誤答案做對照
- **CAD 場景感知對比解碼**
  - RAG 場景：移除檢索文檔生成錯誤輸出
  - 影像場景：加雜訊、打亂畫面、遮蓋關鍵物體
  - 語音場景：靜音、雜音替換音訊輸入
- **MTI 最小測試時介入**
  - 核心：利用 KV Cache 減少對比運算
  - 僅追加少量錯誤提示 Token，避免重複計算
  - 關鍵 Token 選擇：高熵、模型決策模糊位置
- 多維度修正位置研究
  - Logits 層、隱藏層 Representation、Attention 權重皆可做對比修正

## 🔹 方向二：修改 Workflow 執行流程
- 核心架構：**Generation + Verification 生成+自主驗證**
- 實現方式：程式自動插入通用反思提示詞，無人工介入
### 1. 反思機制有效性直覺解釋
- 批判難度低於生成，模型回頭檢查更易發現錯誤
- 自動生成無法回溯，單步錯誤會連鎖偏離
- 海量語料慣性：反思提示觸發文本修正慣性行文

### 2. 實驗結論與侷限
- 反思效果不穩定：強大基座模型提升幅度極小
- 外部回饋（檢查清單、規範約束）遠比自我反思有效
### 3. 模型自我修正四種場景
- 錯→對：核心價值場景
- 對→對：純耗費算力
- 錯→錯：無實質影響
- 對→錯：最嚴重負面效果

### 4. 兩大關鍵指標
- CL 信心指標：正確答案保留率，模型固執程度
- CS 批判指標：錯誤答案修正率，模型彈性程度
- 模型個性分化：頑固模型 VS 易修正模型互斥

### 5. 反思提示詞的影響
- 肯定式提示：提升信心、降低修正意願
- 質疑式提示：降低信心、提高錯誤修正率
- 文獻結論矛盾根源：提示詞差異+模型屬性差異

### 6. 算力性价比對比
- 同等算力下：多輪隨機採樣投票 > 自我反思
- 反思屬於「高投入微提升」的奢侈品優化
- 適用場景：基礎投票優化達瓶頸後再導入驗證流程

## 🔹 方向三：修改模型參數｜內在 Reasoning 推理
- 核心目標：去掉手動反思提示，模型內化自省修正能力
### 1. 核心核心結論
- 知識儲備 ≠ 自我修正能力
- 修正是獨立可抽離的狀態（Steering Vector 導向向量）

### 2. 分階段訓練：REVISE 框架
- 階段1：錯誤偵測訓練，錯答輸出 Refine Token
- 階段2：錯誤修正訓練，基於錯答+Refine 生成正解
- 局限：微調後模型錯誤分布改變，泛化修正能力變弱

### 3. RL 強化學習實現修正
- RLVR 可驗證獎勵強化學習
- 場景：數學、程式等答案客觀可驗證任務
- 機制：僅以最終正解給予獎勵，自發演化驗證+修正流程

### 4. Reasoning 機制底層邏輯
- 拆解複雜步驟，降低單步學習難度
- 減少訓練數據依賴，提升泛化能力
- 直觀案例：二進制奇偶校驗，分步 XOR 遠比一次性判斷簡單

### 5. 核心辯論：RL 是否創造新能力
- 觀點A（內建能力論）
  - 基礎模型原本存在正確推理路徑
  - RL 僅提高正確路徑採樣機率
  - 高次採樣下，有無 RL 效果趨同
- 觀點B（能力進化論）
  - RL 學會全新推理步驟與運算邏輯
  - 約束推理過程正確性，而非僅答案正確
- 整合結論：訓練初期強化現有路徑，後期誕生全新能力

## 📎 整體總結
- 三層實現路徑：推論層優化 → 流程層優化 → 參數層訓練
- 對比解碼：低成本無訓練方案，適合快速落地
- 流程反思：落地簡單、性價比有限，依賴提示調校
- 推理訓練：長期核心方向，決定模型原生自省上限
- 前沿爭議：LLM 自我修正為領域熱門，邊界與機制仍在研究
